\section{个人思考}

\subsection{最值得吸收的思想}

WildDet3D 最有启发的地方不是某一个单独损失，而是系统级设计思路：把开放世界 3D 检测拆成提示条件化、2D 查询生成、几何增强和 3D 提升几个相对清晰的模块。这样做的好处是每个模块都有明确职责，后续替换深度后端、视觉 backbone 或提示编码器时，不必推倒整个系统。

第二个值得借鉴的点是“可选深度”而不是“强制深度”。很多 3D 方法要么只做单目 RGB，要么假设深度或 LiDAR 一定可用。WildDet3D 的设定更贴近真实部署：手机、网页图片、机器人、AR 设备拥有的传感器条件不同，同一模型最好能按输入条件退化或增强。

第三个值得关注的是 O2M 匹配。它解决的是 query-based 检测中正样本监督稀疏的问题。一个 GT 对 top-k query 同时施加监督，相当于让更多候选在训练早期获得有效梯度；这对 promptable detector 和 3D head 的联合训练尤其重要。

\subsection{需要警惕的问题}

这篇方法仍然无法从根本上消除单目 3D 的不可观测性。没有真实深度时，模型的几何判断主要依赖数据先验；一旦遇到长尾类别、罕见姿态、强遮挡或尺度异常的目标，3D 尺寸和深度仍可能不稳定。

此外，开放世界数据的标注质量非常关键。论文通过 ignore region 减少非穷尽标注造成的伪负例，但这只能缓解问题，不能完全解决数据缺失、类别边界模糊和 3D 标注噪声。若把该思路迁移到自己的任务，需要优先检查数据是否存在大量“看得见但没标 3D”的对象。

\subsection{复现与迁移建议}

\begin{enumerate}
    \item 先复现 2D query 与 promptable detector，而不是直接从 3D loss 开始。因为消融显示 2D head 对整体性能影响最大。
    \item 单独验证 RGB-only、完整深度和残缺深度三种输入条件，确认模型确实学会了 optional depth，而不是只在理想深度输入下有效。
    \item 实现 O2M matching 时重点检查 $M$、$N_{pos}$、ignore region 和负例集合，匹配逻辑错误会直接污染所有后续损失。
    \item 如果迁移到自己的课题，可优先借鉴“提示统一编码 + query 生成 + 几何后端可插拔”的框架，而不必完整复刻所有训练损失。
\end{enumerate}
